## 一、项目概览

**通心 (Teacher Hub)** 是一款面向金融培训与社交的跨平台应用，支持 Android、iOS、Web、Windows、macOS、Linux。核心功能包括：行情查看、交易员关注、即时通讯、语音/视频通话、个人中心等。

| 项目 | 说明 |
|------|------|
| 前端 | Flutter 3.x（tongxin-frontend） |
| 后端 | Node.js + Express（tongxin-backend，行情代理） |
| BaaS | Supabase（实时、存储、RLS） |
| 认证 | Firebase Auth（邮箱、Google、Apple） |
| 通话 | Agora RTC Engine |
| 版本 | 0.1.1+2 |

## 二、代码质量检查

### 2.1 静态检查

- **Linter**：`lib/` 目录下无 linter 报错，代码规范良好
- **依赖**：`pubspec.yaml` 依赖版本明确，使用 `^` 语义化版本
- **环境**：SDK 约束 `>=3.0.0 <4.0.0`，符合 Flutter 3.x 要求

### 2.2 架构与设计

| 方面 | 评价 | 说明 |
|------|------|------|
| 模块划分 | 良好 | 按 `features/` 划分（home、market、messages、teachers、trading、call、admin 等），职责清晰 |
| 响应式布局 | 良好 | 宽度 ≥1100px 使用 `PcShell` 桌面布局，否则使用底部导航 |
| 错误处理 | 良好 | `NetworkErrorHelper` 统一处理网络/认证/权限错误，提供友好提示 |
| 本地化 | 良好 | 支持中英文（app_en.arb / app_zh.arb），`LocaleProvider` 管理语言切换 |
| 用户限制 | 良好 | `UserRestrictions` 支持封禁、冻结、限制登录/发消息/加好友等，带缓存 |

### 2.3 启动与生命周期

- **启动流程**：`main.dart` → dotenv → Firebase/Supabase 初始化 → `TvOrbitSplash` 启动页 → `HomePage`
- **Web 无 Firebase**：直接进入 `AdminWebApp` 管理后台
- **生命周期**：`WidgetsBindingObserver` 监听前后台切换，处理通知 Token 保存、未读角标刷新、来电接听/拒绝检查、最后上线时间更新

### 2.4 潜在改进点

1. **app.dart 缩进**：第 90 行 `);` 缩进略不一致，可统一格式
2. **启动页文案**：`TvOrbitSplash` 默认 `title: 'tufei'`、`subtitle: 'Professional · Connected · Forward'`，建议改为应用品牌或可配置
3. **.env 敏感信息**：`.env` 含 API Key，需确保 `.gitignore` 已排除，避免泄露

## 三、功能模块体验

### 3.1 导航与布局

| 端 | 布局 | 入口 |
|----|------|------|
| 移动端 | 底部 5 Tab | 首页、行情、关注、消息、个人 |
| 桌面端 | 侧栏 6 项 | 首页、行情、自选、消息、排行榜、个人 |

- 桌面端使用 `PcShell` + `PcSidebar` + `PcTopbar`，内容区支持 `Navigator` 子页导航
- 首页 Dashboard 可跳转到各子模块，导航逻辑清晰

### 3.2 行情模块

- 支持美股、外汇、加密货币
- K 线、涨跌榜、自选、搜索
- 数据源：Polygon.io、Twelve Data（可经后端代理或直连）
- 后端缓存：内存 → Supabase → SQLite，支持 SingleFlight、限流、后台刷新

### 3.3 消息与社交

- 私聊、群聊、好友、客服工作台、系统通知
- Supabase Realtime 实时消息
- 好友请求订阅、未读角标
- 用户限制（发消息、加好友、加群、建群）与后台配置联动

### 3.4 通话

- Agora 语音/视频
- 来电弹窗、邀请流程
- Edge Function 生成 Token

### 3.5 管理后台

- Web 端独立入口（无 Firebase 时）
- 教师管理、举报、设置
- 与主应用共用主题（深色、金色主色）

## 六、安全与配置

### 6.1 认证与权限

- Firebase Auth 与 Supabase 通过 Firebase ID Token 打通
- RLS 使用 `auth.uid()` 与 Firebase UID 一致
- `UserRestrictions` 支持封禁、冻结及细粒度限制

### 6.2 配置要求

- 前端 `.env`：SUPABASE_URL、ANON_KEY、TONGXIN_API_URL、POLYGON/TWELVE_DATA_API_KEY、AGORA_APP_ID
- 后端：PORT、POLYGON_API_KEY、TWELVE_DATA_API_KEY、SUPABASE_URL、SUPABASE_ANON_KEY
- 不配置 `TONGXIN_API_URL` 时，前端可直连 Polygon/Twelve Data（需在 .env 配置对应 Key）
