# 完整看盘方案：美股 + 全球指数 + 外汇 + 加密货币

目标：**一个能正常看盘的行情系统**，包含美股、全球指数、外汇、加密货币；具备**行情、成交量、K 线、分时图**等该有的全部能力。

---

## 一、你需要的数据清单（该有的都必须有）

| 类型 | 必需能力 | 说明 |
|------|----------|------|
| **行情** | 最新价、涨跌、涨跌幅、前收 | 列表与详情页展示 |
| **成交量** | 当日量、单根 K 线量 | 列表 Volume 列、K 线/分时副图 |
| **K 线** | 日 K / 周 K / 月 K，OHLC + Volume | 蜡烛图、周期切换 |
| **分时图** | 分钟级 OHLC（如 1 分钟）+ Volume | 当日或近几小时走势 |
| **美股** | 个股 + 领涨/领跌榜 | 列表、详情、图表 |
| **全球指数** | 道琼斯、纳斯达克、标普、恒生、日经等 | 概况页、指数详情与图表 |
| **外汇** | 主要货币对（如 EUR/USD、USD/JPY） | 行情 + K 线 + 分时 |
| **加密货币** | BTC、ETH 等主流币 | 行情 + K 线 + 分时 |

---

## 二、推荐方案：一个 API 全覆盖

要**一个 Key 搞定美股 + 全球指数 + 外汇 + 加密货币**，且都带**行情、成交量、K 线、分时**，优先选下面两个之一。

### 方案 A：Twelve Data（推荐优先试）

- **官网**：https://twelvedata.com  
- **文档**：https://twelvedata.com/docs  

| 能力 | 支持情况 |
|------|----------|
| 美股 | ✅ 实时/历史行情，OHLCV，1min/5min/15min/1h/1day 等 |
| 全球指数 | ✅ 多交易所指数，同一套 OHLCV 接口 |
| 外汇 | ✅ 140+ 货币、2000+ 货币对，OHLCV |
| 加密货币 | ✅ 主流币种，OHLCV |
| 成交量 | ✅ 股票/指数有 volume；外汇/ crypto 部分有 |
| K 线 | ✅ 多周期（1min～1month） |
| 分时 | ✅ 1min、5min 等 intraday |
| 免费档 | ✅ 有，限 8 次/分、800 次/天，可先对接再升级 |

**接口风格**：  
- 时间序列（K 线/分时）：`GET /time_series?symbol=AAPL&interval=1min&apikey=xxx`，返回 OHLCV。  
- 实时报价：`GET /quote?symbol=AAPL&apikey=xxx`。  
- 外汇/指数/加密货币：换 `symbol` 即可（如 `EUR/USD`、`BTC/USD`、指数代码见文档）。

适合：**一个 API 覆盖全部资产**，你现有 Polygon 可继续做美股主力，Twelve Data 补指数/外汇/加密货币；或直接用 Twelve Data 做全市场。

---

### 方案 B：EODHD（EOD Historical Data）

- **官网**：https://eodhistoricaldata.com  
- **文档**：https://eodhd.com/financial-apis  

| 能力 | 支持情况 |
|------|----------|
| 美股 | ✅ 实时 WebSocket + REST，OHLCV，预/盘后 |
| 全球指数 | ✅ 600+ 指数，多交易所 |
| 外汇 | ✅ 1100+ 货币对 |
| 加密货币 | ✅ 主流币，OHLCV |
| 成交量 | ✅ 股票/指数/部分 forex 与 crypto |
| K 线 / 分时 | ✅ 历史 + 当日 intraday |
| 价格 | 约 $19.99/月起，有免费试用 |

适合：需要**历史数据更深、全球股票更多**时选用。

## 三、和当前 Polygon 的搭配方式

你现在已有 **Polygon.io**（美股行情、领涨/领跌、K 线、分时、WebSocket）。

- **选项 1：Polygon（美股）+ Twelve Data（其余）**  
  - 美股：继续用 Polygon（领涨/领跌、实时、K 线、分时、成交量）。  
  - 全球指数、外汇、加密货币：用 Twelve Data 的 `time_series` + `quote`，统一成「行情 + 成交量 + K 线 + 分时」的模型，在概况/指数/外汇/加密货币 Tab 里展示。  

- **选项 2：全部用 Twelve Data**  
  - 美股、全球指数、外汇、加密货币都走 Twelve Data，统一接口、统一图表逻辑；领涨/领跌可用其「涨跌幅排序」或类似接口（见文档）。  

- **选项 3：全部用 EODHD**  
  - 同上，但用 EODHD 的 REST + WebSocket 覆盖全部资产。

## 五、实现时「该有的都必须有」检查表

对**每一种资产**（美股、指数、外汇、加密货币），在列表和详情页确保：

- [ ] **行情**：最新价、涨跌、涨跌幅（及前收 where 适用）
- [ ] **成交量**：列表有「成交量」列；详情有「当日量」或「当前周期量」
- [ ] **K 线图**：日 K（及可选周 K/月 K），OHLC + 成交量柱状图
- [ ] **分时图**：分钟级（如 1min）OHLC 或 close 连线 + 成交量副图
- [ ] 数据源统一：要么全从 Twelve Data 来，要么 Polygon（美股）+ Twelve Data（指数/外汇/加密货币），避免混用导致字段不一致

按上表对接后，即可做到：**一个能正常看盘的行情系统，美股、全球指数、外汇、加密货币都有行情、数据、成交量、K 线、分时图，该有的都有**。

如需在项目里**直接加 Twelve Data 的封装与调用示例**（例如 `TwelveDataRepository` + 概况页指数/外汇/加密货币接口），可以说明你希望先接哪一块（指数 / 外汇 / 加密货币），我按那一块写具体接口与字段映射。
