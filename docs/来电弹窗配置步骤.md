# 被叫来电弹窗：一步一做配置

按下面顺序做，做完一步再做下一步。

---

## 第一步：在 Supabase 里添加 Firebase 集成

1. 浏览器打开：**https://supabase.com/dashboard**，登录。
2. 点击你的项目（teacher_hub 用的那个，例如 theqizksqjrylsnrrrhx）。
3. 左侧菜单点 **Authentication**（认证）。
4. 在 Authentication 下找到 **Providers** 或 **Third-party Auth**（第三方认证），点进去。
   - 若没有「Third-party Auth」，看有没有 **Providers** → 里面是否有 **Firebase** 或「添加更多」。
   - 新版本可能在：**Project Settings**（项目设置）→ **Auth** → **Third-party Auth**。
5. 点 **Add integration** / **添加集成**（或类似按钮）。
6. 在列表里选 **Firebase**。
7. 唯一必填：**Firebase Project ID**。先不填，去做第二步拿到后再回来填。

---

## 第二步：拿到 Firebase 项目 ID

1. 浏览器新开一个标签，打开：**https://console.firebase.google.com**。
2. 选中你的项目（和 App 里用的 `google-services.json` 对应的那个）。
3. 点左上角 **齿轮图标** → **项目设置**。
4. 在「常规」页里找到 **项目 ID**（一串英文/数字，例如 `teacher-hub-xxx`）。
5. **复制**这段项目 ID（Ctrl+C）。

---

## 第三步：把 Firebase 项目 ID 填回 Supabase

1. 回到 Supabase 的 Firebase 集成页面（第一步的页面）。
2. 在 **Firebase Project ID** 框里 **粘贴**刚才复制的项目 ID。
3. 点 **Save** / **保存**。

---

## 第四步：确认 call_invitations 表开了 Realtime

1. 在 Supabase 控制台，左侧点 **Database**。
2. 找到 **Replication** 或 **Realtime**（有的版本在 Database 下的子菜单）。
3. 在表列表里找到 **call_invitations**。
4. 把 **call_invitations** 右侧的开关打开（启用 Realtime 发布）。
5. 若没有这个表，说明还没执行 `docs/call_invitations_schema.sql`，需要先执行该 SQL 建表。

---

## 第五步：给现有 Firebase 用户加上 role（让 Supabase 认人）

需要跑一次本地脚本，给当前所有 Firebase 用户加上 `role: 'authenticated'`。

1. 在电脑上新建一个空文件夹，例如：`d:\firebase-claim`。
2. 打开 PowerShell，执行：
   ```powershell
   cd d:\firebase-claim
   npm init -y
   npm install firebase-admin
   ```
3. 去 Firebase 控制台拿「服务账号密钥」：
   - https://console.firebase.google.com → 你的项目 → 齿轮 **项目设置** → **服务账号**。
   - 下面选 **Firebase Admin SDK**，点 **生成新的私钥**，下载 JSON。
   - 把下载的 JSON 放到 `d:\firebase-claim` 文件夹里，改名为 **serviceAccountKey.json**。
4. 在 `d:\firebase-claim` 里新建文件 **set-role-claim.js**，内容如下（整段复制）：

```javascript
'use strict';
const admin = require('firebase-admin');
const path = require('path');

admin.initializeApp({ credential: admin.credential.cert(require(path.join(__dirname, 'serviceAccountKey.json'))) });
const auth = admin.auth();

async function setRoleForAll() {
  let nextPageToken;
  let total = 0;
  do {
    const list = await auth.listUsers(1000, nextPageToken);
    nextPageToken = list.pageToken;
    await Promise.all(list.users.map(async (u) => {
      await auth.setCustomUserClaims(u.uid, { role: 'authenticated' });
      total++;
      console.log('OK', u.uid);
    }));
  } while (nextPageToken);
  console.log('Done. Total users set:', total);
}

setRoleForAll().then(() => process.exit(0)).catch((e) => { console.error(e); process.exit(1); });
```

5. 在 PowerShell 里执行：
   ```powershell
   cd d:\firebase-claim
   node set-role-claim.js
   ```
   看到 `Done. Total users set: N` 就表示现有用户都加好了。

---

## 第六步：两台手机都「退出登录 → 再登录」

1. **主叫手机**：打开 App → 进入「我的」→ 设置里退出登录，再重新登录。
2. **被叫手机**：同样退出登录，再重新登录。

这样两台手机都会拿到带 `role: 'authenticated'` 的新 Token，Supabase 才能正确识别用户，被叫才能收到来电。

---

## 第七步：再测一次通话

1. 被叫手机：打开 App，停留在首页或任意页（不要关掉 App）。
2. 主叫手机：进和对方的单聊，点右上角电话图标 → 选「语音通话」或「视频通话」。
3. 被叫手机应弹出「xxx 邀请你语音/视频通话」的对话框，点「接听」或「拒绝」测试。
