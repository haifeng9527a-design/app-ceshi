# 交易员账户资金逻辑分析

## 一、您提出的公式分析

### 1. 可用资金公式
> 可用资金 = 账户资金 - 委托挂单冻结 - 买入成交资金 - 合约保证金

**分析**：方向正确，但表述可更精确：
- **现货买入**：买入成交时直接扣减 `cash_balance`，`cash_available` 已不含这笔钱
- **合约开仓**：开仓时占用保证金，从 `cash_available` 转为 `used_margin`
- **委托冻结**：下单时 `cash_available` 减少、`cash_frozen` 增加

**正确关系**：
```
可用资金 = 总现金 - 冻结资金 - 已用保证金
即：cash_available = (cash_available + cash_frozen + used_margin) - cash_frozen - used_margin
```

### 2. 总资产公式
> 总资产 = 冻结资金 + 所有持仓盈亏 + 总市值

**分析**：该公式**不完整**，缺少两项关键组成部分：
- 缺少 **可用资金**：账户里可自由使用的现金
- 缺少 **已用保证金**：合约持仓占用的保证金（仍属于你的资产）

**完整公式应为**：
```
总资产 = 可用资金 + 冻结资金 + 已用保证金 + 现货持仓市值 + 合约未实现盈亏
```

## 二、当前系统实现

### 1. 总资产（equity）计算公式

**后端** `apiTrading.js` 与 **Supabase** `get_teacher_trading_summary` 均使用：

```
equity = cash_available + cash_frozen + used_margin + spot_market_value + contract_unrealized
```

| 组成部分 | 含义 |
|---------|------|
| cash_available | 可用资金 |
| cash_frozen | 委托冻结资金 |
| used_margin | 合约已用保证金 |
| spot_market_value | 现货持仓市值（仅现货） |
| contract_unrealized | 合约未实现盈亏（仅合约） |

**结论**：总资产公式在逻辑上是**正确的**。

### 2. 可用资金（cash_available）维护

- **现货买入**：`cash_balance -= 成本`，`cash_available` 相应减少
- **现货卖出**：`cash_balance += 收入`，`cash_available` 增加
- **合约开仓**：从冻结释放后，`cash_available += refund`，`used_margin += requiredMargin`
- **委托挂单**：`cash_available -= frozen`，`cash_frozen += frozen`

**结论**：可用资金维护逻辑**正确**。

## 三、发现的问题

### 问题 1：界面「市值」展示易误导

**现象**：您截图中「市值」显示为 **2,993,268.52**，而「总资产」为 **1,270,078.12**，市值大于总资产。

**原因**：当前「市值」对所有持仓（含合约）按 `价格 × 数量 × 合约乘数` 累加，对合约而言这是**名义价值（notional）**，不是实际投入的资产。

- **现货**：市值 = 持仓数量 × 当前价，即真实持仓价值
- **合约**：名义价值 = 杠杆后的名义规模，实际投入仅为保证金

例如：10 手 BTC 合约 @ 68000，名义价值约 68 万，但 5 倍杠杆下保证金约 13.6 万。把 299 万显示为「市值」会让人误以为有 300 万资产。

**建议**：
1. 将「市值」拆分为「现货市值」和「合约名义价值」，或
2. 仅对现货持仓显示「市值」，合约单独显示「保证金」和「未实现盈亏」

### 问题 2：cash_balance 与合约场景

**现象**：合约开仓时只更新 `cash_available`、`used_margin`，不更新 `cash_balance`。

**影响**：对纯合约账户，`cash_balance` 可能长期等于初始资金，与真实资金状态不一致。但**总资产计算不依赖 `cash_balance`**，因此不影响 equity 正确性。

## 五、总结与建议

1. **总资产公式**：当前实现正确，无需修改。
2. **可用资金**：逻辑正确，与「账户资金 - 冻结 - 保证金」一致。
3. **需优化**：界面「市值」的统计与展示方式，避免把合约名义价值混入「市值」，造成误解。

建议在 UI 上区分：
- **现货市值**：仅现货持仓的市值
- **合约保证金**：已用保证金
- **合约未实现盈亏**：浮动盈亏
- **总资产**：上述各项与现金部分的加总
