# market_page.dart UI 分析与专业行情界面改造方案

## 一、当前 UI 结构分析

### 1.1 整体架构

```
MarketPage (Scaffold)
├── SafeArea
│   ├── _buildHeader()          // 标题「市场」+ 4 个 IconButton（AI / 自选 / 搜索 / 邮件）
│   ├── TabBar                  // 5 个 Tab：概况 | 港股 | 美股 | 外汇 | 加密货币
│   └── TabBarView
│       ├── _OverviewTab        // 概况：指数/外汇/加密货币 + 地图 + 资讯
│       ├── _HKStocksTab        // 港股：占位「敬请期待」
│       ├── _UsStocksTab        // 美股：领涨/领跌榜 + 表格
│       ├── _ForexTab           // 外汇：列表
│       └── _CryptoTab          // 加密货币：三卡 + 热点 + 可交易列表
```

### 1.2 各模块现状

| 模块 | 当前实现 | 问题/缺口 |
|------|----------|------------|
| **市场首页** | 即「概况」Tab：地图概览 + 环球指数 2x 网格 + 资讯 + 外汇/加密货币网格 | 首页与「概况」混在一起；无统一「首页」入口；地图为占位图标；资讯为静态文案 |
| **自选股** | 仅 Header 星形图标 → 跳转独立 WatchlistPage | 自选未在首页露出；无快捷入口在 Tab 内；无自选预览条 |
| **涨跌榜** | 仅在「美股」Tab 内：领涨/领跌 Chip + 单表格 | 无独立涨跌榜入口；港股/外汇/加密货币无统一涨跌榜形态；表格列固定，无排序/筛选 |
| **搜索** | 仅 Header 搜索图标 → 跳转 SearchPage | 搜索未与列表联动；无「搜索框常驻」或「下拉即搜」；无热门/历史建议 |

### 1.3 技术细节

- **数据**：概况/外汇/加密货币用 `MarketRepository.getQuotes`；美股用 `getTopGainers/getTopLosers`；自选用 `WatchlistRepository`；搜索用 `MarketRepository.searchSymbols`。
- **样式**：深色主题（0xFF0B0C0E 背景、0xFFD4AF37 强调、0xFF111215 卡片）；列表多为 `ListView`/`GridView`，美股为横向滚动表格（minWidth 560）。
- **导航**：详情统一 `Navigator.push`（StockChartPage / GenericChartPage），无命名路由。

---

## 二、改造目标（专业行情软件对标）

参考同花顺/东方财富/MOMO 等：

1. **市场首页**：一眼看到「大盘/指数 + 自选摘要 + 涨跌榜入口 + 资讯/快讯」的仪表盘。
2. **自选股**：独立 Tab 或首页首屏区块，支持编辑、拖拽排序、分组，与行情列表样式统一。
3. **涨跌榜**：独立入口，支持多市场（美股/港股/外汇/加密货币）、多维度（涨跌幅/成交量/换手等）、可配置列与排序。
4. **搜索**：常驻或轻量入口，支持代码/拼音/名称，结果可加自选、直接进详情，带热门与历史。

## 三、改造方案（按模块）

### 3.1 市场首页

**思路**：将「首页」与「概况」区分；首页作为默认第一屏，突出核心信息与入口。

| 项目 | 当前 | 改造方案 |
|------|------|----------|
| **入口** | 第一 Tab 为「概况」 | 第一 Tab 改为「**首页**」，原「概况」内容并入首页或作为首页上半部分 |
| **首屏结构** | 地图 + 指数网格 + 资讯 + 外汇 + 加密货币 | **方案 A**：顶部「大盘指数条」（道琼斯/标普/纳指/恒生等 4–6 个，横向滚动，带迷你图）+ 下方「自选预览」（最多 6 条，左滑更多→自选页）+ 再下「领涨/领跌快捷入口」+ 资讯/快讯区 + 外汇/加密货币收折或第二屏。<br>**方案 B**：保留现有概况布局，仅在顶部增加「自选预览条」与「涨跌榜入口」两行。 |
| **大盘指数条** | 概况里为 2 列网格 | 改为单行横向 `ListView`，每项：名称 + 点数 + 涨跌幅 + 迷你 K 线/折线，点击进详情；数据仍用 `MarketRepository.getQuotes`（指数 symbol 列表）。 |
| **地图** | 占位图标 + 指数标签 | 可选：保留为装饰；或改为「全球市场热力图」占位（按区域色块表示涨跌）；或移除，用指数条替代。 |
| **资讯** | 静态标题列表 | 保留区块，后续接真实资讯 API；样式可改为卡片+时间，或一行快讯滚动。 |

**推荐**：采用方案 A，新增「首页」Tab，首屏自上而下：大盘指数条 → 自选预览 → 领涨/领跌入口 → 资讯 → 外汇/加密货币（可折叠）。

---

### 3.2 自选股

**思路**：在 Market 内既有独立自选页，也在首页有曝光与快捷编辑。

| 项目 | 当前 | 改造方案 |
|------|------|----------|
| **入口** | 仅 Header 星形 → WatchlistPage | **保留**；增加：首页「自选预览」区块（显示前 N 条），点击「更多」或星形进入 WatchlistPage。 |
| **Tab 结构** | 无自选 Tab | **可选**：将 Tab 调整为「首页 | 自选 | 美股 | 港股 | 外汇 | 加密货币」，自选 Tab 内嵌与 WatchlistPage 相同的列表（或直接复用 WatchlistPage 的 body 作为 Tab 子组件），避免重复实现。 |
| **自选预览（首页）** | 无 | 首页内一块区域：标题「自选」+ 从 `WatchlistRepository.getWatchlist()` + `MarketRepository.getQuotes(symbols)` 拉取报价；列表 4–6 条，左滑或「全部」进入 WatchlistPage；每条：symbol、最新价、涨跌幅、可滑删或点进详情。 |
| **WatchlistPage** | 列表 + 移除 + 点击详情 | 增强：① 支持长按拖拽排序并写回 `WatchlistRepository`（需持久化顺序）；② 可选分组（如「美股」「港股」）；③ 列表样式与美股涨跌榜表格对齐（代码、最新、涨跌、涨跌幅、成交量等列），便于与专业软件一致。 |

**推荐**：首页增加「自选预览」；Tab 是否增加「自选」视产品优先级，若加则与 WatchlistPage 共用数据与列表 UI。

---

### 3.3 涨跌榜

**思路**：统一「涨跌榜」形态，支持多市场与多维度，入口明显。

| 项目 | 当前 | 改造方案 |
|------|------|----------|
| **入口** | 仅在美股 Tab 内 | **首页**增加「领涨」「领跌」两个入口（按钮或卡片），点击进入「涨跌榜」页或展开涨跌榜区块；**美股 Tab** 保留现有领涨/领跌表格作为「美股涨跌榜」子视图。 |
| **涨跌榜页** | 无独立页 | 新建 **GainersLosersPage**（或同一页 Tab）：顶部市场切换（美股/港股/外汇/加密货币），第二行维度切换（涨跌幅/成交量等，按数据源能力），下方为表格；美股数据用现有 `getTopGainers/getTopLosers`，其他市场用 `getQuotes` 后排序或后续接口。 |
| **表格** | 美股：固定列 #/代码/最新/涨跌/涨跌幅/成交量 | 保持列含义；**增强**：① 表头支持点击排序（涨跌幅/成交量等）；② 可选列（如振幅、换手，依赖 API）；③ 样式与自选/首页列表统一（行高、色块、分割线）。 |
| **港股/外汇/加密货币** | 各自为列表或网格，无「涨跌榜」形态 | 统一为「列表 + 按涨跌幅排序」的涨跌榜视图；数据用现有 `getQuotes` 或 getCryptoQuotes 后排序；若暂无单独接口，先做「客户端排序」涨跌榜。 |

**推荐**：首页增加领涨/领跌入口；新建 GainersLosersPage，先支持「美股」+ 领涨/领跌 Tab，表格可排序；其他市场用同一页面切换市场后，列表按涨跌幅排序展示。

---

### 3.4 搜索

**思路**：搜索常驻或一键可达，结果可加自选、进详情，并带建议与历史。

| 项目 | 当前 | 改造方案 |
|------|------|----------|
| **入口** | Header 搜索图标 → SearchPage | **保留**；可选：首页顶部增加「搜索框占位」（点击即跳 SearchPage 或底部弹出搜索栏），形成「搜索常驻」感。 |
| **SearchPage** | 输入框 + `searchSymbols` + 列表（symbol/name/market）+ 加自选 | 保持主流程；**增强**：① 无输入时展示「热门搜索」或「最近搜索」（最近搜索用 `shared_preferences` 存 symbol 列表）；② 结果行增加「加自选」状态（已自选显示勾选）；③ 支持从结果直接进详情（已有），并可选「加入自选后返回」的快捷操作。 |
| **与列表联动** | 无 | 可选：在涨跌榜/自选列表顶部放「搜索」图标，点击跳转 SearchPage 并预填当前市场或空。 |

**推荐**：SearchPage 增加「最近搜索」与「热门」（可配置静态列表）；结果行已自选时显示勾选；首页可选轻量搜索入口。

## 四、信息架构与导航建议

### 4.1 Tab 结构两种方案

| 方案 | Tab 顺序 | 说明 |
|------|----------|------|
| **A** | 首页 \| 自选 \| 美股 \| 港股 \| 外汇 \| 加密货币 | 首页与自选前置，符合「先看大盘与自选」习惯；Tab 较多，可考虑部分收起到「更多」。 |
| **B** | 首页 \| 行情（子 Tab：美股/港股/外汇/加密货币）\| 自选 | 行情收成一个 Tab，内里再子 Tab；自选独立；首页更突出。 |

当前为 5 Tab（概况/港股/美股/外汇/加密货币），若改为「首页 + 自选 + 行情子 Tab」，建议：

- **方案 A 精简**：`首页 | 自选 | 美股 | 外汇 | 加密货币`（港股仍占位或合并到「行情」子 Tab）。
- 或保持 5 Tab，仅把第一项改为「首页」，内容按 3.1 节重组。

### 4.2 统一组件与样式

- **列表行**：自选预览、涨跌榜、外汇/加密货币列表，统一为「左侧名称+代码，右侧价格+涨跌幅」，可选统一行高、分割线、点击态。
- **表格**：美股涨跌榜、自选（若做表格式）共用 `_TableHeader` / `_StockRow` 风格，列宽可配置或按内容适配。
- **颜色与字体**：继续使用现有深色主题；涨跌红绿统一（如 0xFF22C55E / 0xFFEF4444）；重要数字加粗、次要灰色。

## 五、实施优先级建议

| 阶段 | 内容 | 说明 |
|------|------|------|
| **P0** | 首页增加「自选预览」+ 大盘指数条 + 领涨/领跌入口 | 不改 Tab 数量，仅在现有「概况」或新「首页」内加区块；自选预览调用 WatchlistRepository + getQuotes。 |
| **P0** | 涨跌榜独立页 GainersLosersPage，美股 + 领涨/领跌，表头可排序 | 从美股 Tab 拆出表格为可复用组件，新页用 Tab（领涨/领跌）或 Segment 切换。 |
| **P1** | 第一 Tab 改为「首页」并落地 3.1 首屏结构 | 大盘条、自选预览、涨跌榜入口、资讯、外汇/加密货币收折。 |
| **P1** | SearchPage 最近搜索 + 热门 + 已自选状态 | 本地存最近 N 条 symbol；热门可写死或配置；加自选后刷新勾选状态。 |
| **P2** | 自选 Tab 或 WatchlistPage 表格化、拖拽排序 | 与涨跌榜表格样式统一；顺序持久化。 |
| **P2** | 涨跌榜多市场（港股/外汇/加密货币）| 数据用 getQuotes/getCryptoQuotes 后排序，同一 GainersLosersPage 切换市场。 |

## 六、小结

- **市场首页**：独立「首页」Tab，首屏 = 大盘指数条 + 自选预览 + 领涨/领跌入口 + 资讯 + 外汇/加密货币（可折叠）。
- **自选股**：首页自选预览 + 保留 WatchlistPage；可选自选 Tab；列表/表格与涨跌榜统一，支持排序与持久化。
- **涨跌榜**：独立 GainersLosersPage，领涨/领跌 + 可排序表格；首页与美股 Tab 均提供入口；后续扩展多市场。
- **搜索**：保留当前入口与 SearchPage；增加最近/热门与已自选状态；可选首页轻量搜索入口。

按 P0 → P1 → P2 分阶段落地，即可在不大改现有数据与路由的前提下，将 `market_page.dart` 升级为更接近专业行情软件的界面结构。
