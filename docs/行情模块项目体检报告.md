## 一、项目技术栈

| 类别 | 选型 | 说明 |
|------|------|------|
| **Flutter 版本** | SDK `>=3.0.0 <4.0.0` | 见 `pubspec.yaml` environment，未锁具体 Flutter 版本 |
| **状态管理** | 无统一框架 | 使用 `StatefulWidget` + `setState`，无 provider / bloc / getx / riverpod |
| **路由方式** | `MaterialApp` 命名路由 + `Navigator.push` | `routes: { '/admin': ... }`，其余多为 `Navigator.push(context, MaterialPageRoute(builder: (_) => SomePage()))` |
| **网络库** | `http`（`package:http`） | 未使用 dio |
| **图表库** | `fl_chart: ^0.66.0` | 用于 K 线、分时、迷你图 |
| **本地存储** | `shared_preferences` + 自建文件缓存 | 全局偏好用 shared_preferences；行情专用为 `TradingCache`（`path_provider` + 文件 JSON 缓存） |

## 三、入口与路由

- **入口**：`lib/main.dart`  
  - 初始化：`dotenv`、Firebase、Supabase、通知等；Web 且 Firebase 未就绪时跑 `AdminWebApp`，否则 `runApp(const TeacherHubApp());`  
  - 主 App 根组件：`lib/app.dart` 中的 `TeacherHubApp`。

- **路由表**（`TeacherHubApp` 内）：  
  - `routes: { '/admin': (_) => const AdminHomePage() }`  
  - 无 `/market`、`/stock/:symbol` 等命名路由；行情相关页面均为在 `MarketPage` 或 `HomePage` 内通过 `Navigator.push(...)` 打开。

- **首页**：  
  - `home: const SplashScreen()` → 约 3.2 秒后 `pushReplacement` 到 **`HomePage`**（`lib/features/home/home_page.dart`）。  
  - `HomePage` 底部 5 个 Tab：**主页（RankingsPage）**、**行情（MarketPage）**、**关注（FeaturedTeacherPage）**、**消息（MessagesPage）**、**我的（ProfilePage）**。  
  - 即：用户可见的「首页」是 `HomePage`，默认选中第一个 Tab「主页」`RankingsPage`；**行情**为第二个 Tab，对应 `MarketPage`。

## 四、行情模块现状

| 功能 | 对应页面/入口 | 文件路径 |

| **市场** | 行情主页，Tab：概况 / 港股 / 美股 / 外汇 / 加密货币 | `lib/features/market/market_page.dart`（`MarketPage`） |

## 五、数据层现状

- **Repository / Service**  
  - **Polygon**：`lib/features/trading/polygon_repository.dart`（`PolygonRepository`）— 美股 last trade、前收、领涨/领跌、K 线/分时 aggregates。  
  - **Twelve Data**：`lib/features/trading/twelve_data_repository.dart`（`TwelveDataRepository`）— 全球指数/外汇/加密货币 quote、time_series（K 线/分时）。  
  - **Supabase 快照**：`lib/features/trading/market_snapshot_repository.dart`（`MarketSnapshotRepository`）— 领涨/领跌 payload 存表 `market_snapshots`，休市或新用户无本地缓存时读取。  
  - 无统一「行情门面」Repository，页面直接依赖上述多个 Repository。

- **WebSocket**  
  - `lib/features/trading/polygon_realtime.dart`（`PolygonRealtime`）：连接 `wss://socket.polygon.io/stocks`，订阅单标的成交流（`T.$symbol`），用于 `StockChartPage` 实时价与成交量。  
  - 无 Finnhub 或其他行情 WebSocket。

- **缓存**  
  - **TradingCache**（`lib/features/trading/trading_cache.dart`）：单例，基于 `path_provider` 的 JSON 文件缓存（目录 `trading_cache`），键值 + 过期时间；用于 Polygon last trade、前收、领涨/领跌列表、aggregates、Twelve Data quote/time_series 等。  
  - 领涨/领跌开市时可回写 Supabase（`MarketSnapshotRepository.saveGainers/saveLosers`），供休市或新设备读取。  
  - 无 Hive/SQLite；`shared_preferences` 用于全局设置，未用于行情数据。

## 六、已接入的数据源

- **Polygon.io**  
  - Base URL：`https://api.polygon.io`  
  - 环境变量：需在 `.env` 中配置 `POLYGON_API_KEY`（不在此文档写出具体 Key）。  
  - 主要 endpoint：  
    - `GET /v2/last/trade/{symbol}` — 最后成交  
    - `GET /v2/aggs/ticker/{symbol}/prev` — 前一交易日收盘  
    - `GET /v2/snapshot/locale/us/markets/stocks/gainers` — 美股领涨  
    - `GET /v2/snapshot/locale/us/markets/stocks/losers` — 美股领跌  
    - `GET /v2/aggs/ticker/{symbol}/range/{multiplier}/{timespan}/{fromMs}/{toMs}` — K 线/分时 bars  
  - WebSocket：`wss://socket.polygon.io/stocks`（认证后订阅 `T.{symbol}`）。

- **Twelve Data**  
  - Base URL：`https://api.twelvedata.com`  
  - 环境变量：需在 `.env` 中配置 `TWELVE_DATA_API_KEY`（不在此文档写出具体 Key）。  
  - 主要 endpoint：  
    - `GET /quote?symbol=...` — 单或批量报价  
    - `GET /time_series?symbol=...&interval=...&outputsize=...` — K 线/分时序列  

- **Supabase**  
  - 表 `market_snapshots`：存领涨/领跌 payload（`type` = gainers/losers），用于休市或无本地缓存时的回退。  

- **Finnhub**  
  - 未接入；当前行情数据仅来自 Polygon + Twelve Data。

## 七、需要补齐的缺口清单

### 必须

1. **行情搜索**：顶部搜索图标目前无实现；需搜索页（或弹窗）+ 标的搜索 API（Polygon/Twelve 等）+ 跳转详情或加入自选。  
2. **自选/关注**：行情侧无自选列表与增删；需自选数据源（本地或 Supabase）、自选列表页、与市场/详情页的「加自选」入口。  
3. **错误与无数据态**：网络失败、API 限流、无数据时需统一提示与重试策略，避免空白或仅 debug 打印。  
4. **API Key 与限流**：Key 仅从 `.env` 读取，需明确生产环境注入方式；限流/配额与降级策略未在代码中显式处理。

### 可选

5. **港股 Tab**：`MarketPage` 有「港股」Tab，当前为占位/模拟；若要做实需港股数据源与对应 Repository。  
6. **详情页增强**：开盘/最高/最低/昨收在模型已存在，在 `StockChartPage` 头部或信息区展示；市盈率/市值等需另接 Ticker Details 或参考数据。  
7. **通知**：行情页铃铛入口未实现；涨跌提醒、系统通知等需设计与后端/推送打通。  
8. **路由与深链**：行情详情无命名路由，不利于深链与统一跳转；可增加如 `/market/stock/:symbol`、`/market/quote/:symbol`。  
9. **状态管理**：若后续自选、主题、多 Tab 状态变多，可考虑引入 provider/riverpod 等，便于共享与测试。

## 八、建议的改造路线图

### 阶段一：MVP（先跑通体验）

- 实现**行情搜索**：搜索页 + Polygon/Twelve 搜索或建议接口，结果可点进 `StockChartPage`/`GenericChartPage`。  
- 实现**自选**：本地存储（如 `shared_preferences` 或沿用 `TradingCache`）存 symbol 列表，自选列表页只读展示，详情页与列表项可「加/删自选」。  
- 统一**无数据/错误**：列表与详情页 loading、空态、错误态与重试按钮。  
- 不动现有数据源与页面结构，仅补齐入口与最小闭环。

### 阶段二：完善（体验与可维护性）

- **路由**：为行情详情增加命名路由（如 `/market/stock/:symbol`），`app.dart` 或独立路由表统一解析。  
- **缓存与离线**：明确 TradingCache 各 key 的 TTL 与 Supabase 回退顺序；可选在无网时仅读缓存并标注「离线数据」。  
- **详情页信息**：在 `StockChartPage` 展示开盘/最高/最低/昨收；可选接入 Ticker Details 展示市盈率/市值。  
- **港股 Tab**：若产品需要，接入港股数据源并替换当前占位内容。  
- **通知入口**：铃铛点击进入「通知设置/行情提醒」占位页，为后续推送预留。

### 阶段三：专业（扩展与架构）

- **数据层抽象**：增加 `MarketRepository` 或 UseCase 层，对 Polygon/Twelve/Supabase 做统一封装，便于多数据源切换与单测。  
- **限流与降级**：API 限流检测、退避重试、以及「仅展示缓存」的降级策略。  
- **状态管理**：若自选、主题、多 Tab 状态膨胀，引入 provider/riverpod，将自选列表、当前选中的 Tab 等提升到共享状态。  
- **专业指标**：K 线副图指标（MA/VOL/MACD/KDJ）已有部分，可扩展更多指标与配置持久化。  
- **Finnhub 等**：若有需求，可接入 Finnhub 等作为补充或备用数据源，通过统一 Repository 抽象切换。
